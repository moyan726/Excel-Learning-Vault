# 用一个销售案例学会 Excel 动态报表：从月份下拉、`SUMIF/SUMIFS` 到条件格式

很多人学 Excel，容易停留在“会几个函数”的阶段，但真正到了工作里，往往需要的是另一种能力：把一张明细表整理成一张能切换条件、能自动汇总、还能一眼看出重点的报表。

这份案例很适合拿来练习这一点。源数据并不复杂，却完整覆盖了一张 Excel 动态报表最核心的能力链路：数据准备、月份筛选、跨表引用、绝对引用、条件聚合、环比计算、容错处理，以及最后的视觉优化。

[插图：完整销售汇总报表完成效果]

这张图适合放在开篇，让读者先看到最终要做出来的样子。整张报表分成三层：顶部是月累计指标，中间是按区域和省份展开的成交情况，底部是按产品期数拆解的成交结构，右上角的月份下拉框会联动整张表。读者在这里最该关注的，不是某一个公式，而是“一个筛选条件驱动全表更新”的整体思路。

## 一、这个案例要做什么，为什么适合练习 Excel 报表

这份工作簿的源数据一共 1657 条，覆盖 2020 年 4 月和 5 月两个自然月。目标不是做一张静态表，而是做一张可以按月份动态切换的销售成交汇总表。

从结果上看，这张报表主要回答三类问题：

- 当前月份整体卖了多少钱、成交了多少客户、客单价是多少
- 各区域、各省份的表现如何，哪些地方增长更快，哪些地方在下滑
- 不同期数产品的成交结构是什么，主力贡献来自哪一类产品

这类案例特别适合练习 Excel 报表，原因在于它把“算”和“看”都放进来了。公式要能算对，报表还要让人看懂；筛选要能联动，版式还要有阅读节奏。这才是更接近真实业务的 Excel 用法。

先看这份案例的数据概况：

| 维度 | 内容 |
| --- | --- |
| 数据记录数 | 1657 条 |
| 月份范围 | 2020 年 4 月、5 月 |
| 区域 | 华东、华南、华西北 |
| 省份/城市 | 杭州、苏州、上海、合肥、南京、广州、深圳、北京、成都、重庆、西安 |
| 期数结构 | 主要为 6 期、12 期、18 期 |
| 核心指标 | 成交金额、成交客户数、客单价、月环比 |

## 二、数据源长什么样：源数据表结构与业务字段说明

报表做得顺不顺，第一步不是急着写公式，而是先把源数据表看懂。

### 1. 基础字段：决定你能分析什么

`源数据` 工作表里最核心的字段如下：

| 字段 | 业务含义 | 在报表中的作用 |
| --- | --- | --- |
| 成交日期 | 每笔交易发生的日期 | 用于提取年份、月份 |
| 销售工号 | 销售人员唯一编号 | 保留明细身份信息 |
| 产品 | 具体产品名称，如“借呗12期”“借呗6期” | 用于产品层分析 |
| 成交客户数 | 单笔记录对应客户数量 | 用于统计客户规模 |
| 成交金额 | 单笔交易金额 | 用于金额汇总、环比和占比分析 |
| 产品类型 | 产品大类 | 为后续扩展分析保留维度 |
| 期数 | 6期、12期、18期等 | 用于产品成交情况区块 |
| 区域 | 华东、华南、华西北 | 用于区域分组 |
| 省份 | 城市/省份维度 | 用于省份明细区块 |
| 小组 / 业务组 / 职务类别 | 组织归属信息 | 便于后续更细分析 |

### 2. 辅助字段：决定你后面的公式会不会轻松

这份案例里真正让报表变简单的，不是后面那些长公式，而是前面补的几列辅助字段：

| 字段 | 公式 | 用途 |
| --- | --- | --- |
| 客单价 | `=E2/D2` | 把“金额”转换成“单客价值” |
| 成交年份 | `=YEAR(A2)` | 为按年分析做准备 |
| 成交月份 | `=MONTH(A2)` | 作为本案例最关键的筛选条件 |

这里最关键的是“成交月份”。因为整个动态报表，几乎所有汇总公式都在围绕这一列做条件匹配。

### 3. 这份数据适合怎么理解

如果把这份表当成业务数据来看，它其实是一张销售明细流水：

- 每一行是一笔成交记录
- 日期告诉你它发生在几月
- 区域和省份告诉你它发生在哪个市场
- 期数告诉你它属于哪种产品结构
- 金额和客户数决定你最终能算出总额、人数和客单价

为了让读者对数据规模更有感觉，这份案例在两个核心月份里的表现大致如下：

| 月份 | 成交金额 | 成交客户数 | 客单价 |
| --- | --- | --- | --- |
| 4 月 | 8,361,888 | 665 | 12,574 |
| 5 月 | 16,154,830 | 1,540 | 10,490 |

这组数据很适合拿来讲环比。因为你能明显看到：5 月的成交金额和成交客户数都在大幅增长，但客单价反而下降了。这正好说明，报表不是只看一个指标，而是要把几个指标放在一起看。

## 三、先把维度准备好：区域、省份和期数为什么要先去重

很多人做报表时，会直接手敲一份省份列表和期数列表。这样做当然也能用，但一旦源数据维度变了，模板就容易失真。更稳妥的做法，是直接从明细数据里提取唯一值。

这份案例里，源数据表右侧额外保留了两块辅助区域：

| 辅助区域 | 内容 | 作用 |
| --- | --- | --- |
| `R:S` 列 | 去重后的“区域-省份”组合 | 生成省份明细的行标签 |
| `U` 列 | 去重后的期数列表 | 生成产品成交情况区块 |

[插图：删除重复值生成区域与省份唯一组合]

这张图放在这里最合适，因为它展示的正是“数据准备”的关键动作。图中左侧是明细表里的区域、省份原始列，顶部红框标出“删除重复值”的入口，右侧则是最终保留下来的唯一组合列表。读者要重点关注的是：这里不是简单删除某一列重复值，而是按“区域 + 省份”这组维度一起去重，这样生成的列表才适合直接作为报表明细区的行标签。

> 小提示：这一步看起来很基础，但特别重要。因为报表行标签一旦准备得稳定，后面的 `SUMIFS` 公式就可以整块下拉复制，不用手工逐行改条件。

在真正开始排版前，还有一个很实用的小技巧：善用名称框快速选区。尤其是像月累计区、标题区、产品区这种连续区域，很多时候不需要鼠标慢慢拖，直接输入区域地址会更快。

[插图：使用名称框快速选中报表区域]

这张图展示的是 Excel 左上角名称框的用法。图中直接输入了 `A6:H7`，回车后就能立刻选中整块区域。它适合放在这里，是因为报表制作进入排版和格式阶段之后，会频繁选中整块区域做边框、填充、对齐和条件格式设置。读者需要重点注意的是名称框的位置，以及“输入区域地址后按回车即可选区”的操作逻辑，这比纯靠鼠标拖拽高效得多。

## 四、如何做一个可切换月份的动态报表

动态报表的控制核心，不在十几个公式，而在一个单元格：月份筛选框。

这份案例把控制单元格放在 `H4`。用户只要改这里的月份，整张报表就会联动更新。

### 1. 用数据验证把 `H4` 变成下拉菜单

最常见的做法是给 `H4` 设置数据验证，允许值为一个固定序列。

```excel
允许：序列
来源：4,5
```

[插图：数据验证下拉菜单设置界面]

这张图展示了数据验证的完整入口和设置细节。读者需要重点看四个位置：顶部“数据”选项卡里的入口按钮、对话框中的“允许”字段、是否勾选“提供下拉箭头”，以及来源框里填入的 `4,5`。这张图放在这里的意义，是让读者真正明白：动态报表不是靠 VBA，也不是靠复杂控件，很多时候一个数据验证下拉框就够了。

为什么这里要做成下拉框，而不是让用户手输数字？因为这样可以限制输入范围，避免把月份写错，也让整个报表的交互更像“成品”而不是“草稿”。

### 2. 标题不要写死，而要跟着月份一起变

这份报表的标题不是手工输入，而是用公式拼出来的：

```excel
=H4&"月销售数据汇总"
```

这个公式不复杂，但很值得学。因为它体现了一种非常实用的报表思路：**凡是会跟筛选条件变化的内容，都尽量不要手写，而要让它自动生成。**

当 `H4` 为 5 时，标题显示为“5月销售数据汇总”；当 `H4` 切回 4 时，标题也会随之变成“4月销售数据汇总”。

## 五、报表里的核心公式：`SUMIF` / `SUMIFS` / 文本拼接 / `IFERROR` / 环比

这一部分是整张报表的计算核心。公式看起来不少，但只要按“公式 -> 拆解 -> 业务含义 -> 易错点”去理解，逻辑其实很清楚。

### 1. 月累计区：先算金额和客户数，再算客单价

顶部月累计区有三个核心指标：成交金额、成交客户数、客单价。

对应公式分别是：

```excel
=SUMIF(源数据!$O:$O,$H$4,源数据!$E:$E)
=SUMIF(源数据!$O:$O,$H$4,源数据!$D:$D)
=B6/D6
```

可以这样理解：

- 第一条公式：到源数据的“成交月份”列里，找出等于当前月份的记录，再把对应“成交金额”加总
- 第二条公式：逻辑一样，只是把求和目标换成“成交客户数”
- 第三条公式：用总金额除以总客户数，得到客单价

这一步解决的是“月度总览”问题。也就是切到某个月时，这个月总体卖了多少、成交了多少人、平均每个客户贡献多少金额。

### 2. 为什么这里先用 `SUMIF`

因为月累计区的条件只有一个：月份。  
当条件只有一个时，`SUMIF` 足够简洁。

```excel
=SUMIF(条件范围, 条件, 求和范围)
```

但需要马上提醒一个常见误区：`SUMIF` 只适合单条件汇总；一旦进入省份明细区、产品期数区，就应该切换到 `SUMIFS`。

### 3. 省份成交情况：一旦有多个条件，就该用 `SUMIFS`

看一条省份成交金额公式：

```excel
=SUMIFS(源数据!$E:$E,源数据!$H:$H,$A11,源数据!$I:$I,$B11,源数据!$O:$O,$H$4)
```

拆开看，它同时满足了三组条件：

| 条件范围 | 条件 | 业务含义 |
| --- | --- | --- |
| `源数据!$H:$H` | `$A11` | 匹配当前行所属区域 |
| `源数据!$I:$I` | `$B11` | 匹配当前行所属省份 |
| `源数据!$O:$O` | `$H$4` | 匹配当前选中的月份 |

满足这三个条件的记录，才会参与金额汇总。

从业务角度看，这条公式回答的是：**某个区域下某个省份，在当前月份一共成交了多少钱。**

### 4. 为什么这里最好把区域条件也写上

严格说，这个案例里只用省份和月份，也能算出结果，因为 11 个省份没有重名问题。但写上区域条件，公式会更稳。

原因很简单：

- 公式逻辑更完整
- 报表结构和公式结构一致，更容易读
- 以后如果维度扩展，出错概率更低

对于博客读者来说，这也是一个很重要的实战习惯：**不是“能算出来就行”，而是“公式要写得经得起扩展”。**

### 5. 环比怎么理解：先讲人话，再写公式

环比本质上只有一句话：

```text
本期 ÷ 上期 - 1
```

放到报表里，成交金额环比可以写成：

```excel
=IFERROR(
 C11/SUMIFS(源数据!$E:$E,源数据!$H:$H,$A11,源数据!$I:$I,$B11,源数据!$O:$O,$H$4-1)-1,
 "暂无数据"
)
```

这个公式的逻辑是：

1. `C11` 是当前月份当前省份的成交金额
2. `SUMIFS(...)` 重新去算上一个月份的成交金额
3. 用“本期 ÷ 上期 - 1”得到环比增长率
4. 用 `IFERROR` 兜底，避免上月无数据时报错

这一步解决的是“趋势”问题。金额本身告诉你规模，环比告诉你变化。

### 6. `IFERROR` 为什么是报表里的必备函数

一旦做动态报表，容错几乎是必需的。因为并不是每一个省份、每一期数、每一个月份组合都一定有数据。

比如这份案例里：

- 南京在 5 月有数据，但 4 月没有
- 1 期、3 期、9 期在报表模板里预留了位置，但当月实际为 0

如果公式里没有 `IFERROR`，这些地方就会出现 `#DIV/0!` 一类的错误值。  
这类错误对于 Excel 来说不是问题，但对于报表读者来说会非常影响阅读。

`IFERROR` 的写法很简单：

```excel
=IFERROR(原公式,"暂无数据")
```

它的意义不是“修正计算”，而是“让报表在异常场景下也保持可读”。

### 7. 客单价是最容易被算错的指标

客单价看起来简单，但恰恰最容易出错。

很多初学者会犯两个典型错误：

1. 直接对客单价列求和
2. 直接拿客单价去算环比，而不回到金额和客户数重新计算

正确思路永远是：

```text
客单价 = 成交金额 ÷ 成交客户数
```

也就是说，客单价不是原始可汇总指标，而是二次计算指标。

所以客单价环比必须写成：

```text
本期客单价 ÷ 上期客单价 - 1
```

而“上期客单价”本身，也必须重新用“上期金额 ÷ 上期客户数”算出来，不能偷懒。

### 8. 产品成交情况：`SUMIFS` 不只是算金额，还能算结构

底部产品成交情况区块，本质上和省份区块是同一套思路，只不过条件从“区域、省份、月份”切换成了“期数、月份”。

例如 6 期成交金额可以写成：

```excel
=SUMIFS(源数据!$E:$E,源数据!$G:$G,$A29,源数据!$O:$O,$H$4)
```

而占比则是在金额基础上继续计算：

```excel
=B29/$B$33
```

这一步解决的是“结构”问题。  
在 5 月里：

- 12 期成交金额约 753.8 万，占比约 47%
- 6 期成交金额约 524.4 万，占比约 32%
- 18 期成交金额约 337.3 万，占比约 21%

这说明 12 期是主力成交结构，6 期次之，18 期也有稳定贡献。

## 六、为什么这里一定要用绝对引用和跨表引用

Excel 公式能不能整块复制、整列下拉，关键往往不在函数本身，而在引用有没有锁对。

### 1. 跨表引用：数据和报表分开，公式才能清楚

这份案例里，明细数据在 `源数据` 工作表，汇总结果在 `报表复现` 或 `销售汇总表` 工作表。

所以公式里会大量出现这样的写法：

```excel
源数据!$E:$E
报表复现!$H$4
```

这就是标准的跨表引用。  
它的好处很直接：

- 明细层和汇总层分开
- 汇总区不会混进原始明细
- 公式结构更容易读

### 2. `$` 不只是语法细节，而是拖拽复制的前提

这份案例里最常见的几种引用方式如下：

| 写法 | 含义 | 为什么这样锁 |
| --- | --- | --- |
| `$H$4` | 锁列也锁行 | 月份筛选框是全表公共参数 |
| `$A11` | 锁列不锁行 | 向下拖动时，区域要跟行号一起变 |
| `$B11` | 锁列不锁行 | 向下拖动时，省份要跟行号一起变 |
| `源数据!$E:$E` | 锁定求和列 | 明细表的金额列不会变 |

如果这里不锁定，会出现很典型的问题：

- `H4` 往下拖后变成 `H5`、`H6`
- 省份条件锁成 `$B$11`，导致整列都引用杭州
- 求和列没锁住，向右复制时从金额列跑到别的列

这也是为什么 Excel 里 `F4` 这么重要。  
它不是一个“快捷键技巧”，而是写复杂公式时的基本动作。

## 七、如何让报表更好读：条件格式、自定义格式、颜色与视觉表达

一张能发布到博客里的 Excel 报表，不该只是“算出来”，还要“看得懂”。

这份案例里，视觉表达做得很有代表性：不同类型的指标，用了不同类型的条件格式。

### 1. 月环比最适合先用图标集表达方向

[插图：为月环比列设置图标集规则]

这张图展示的是“开始 -> 条件格式 -> 图标集 -> 新建规则”的设置方式。最关键的地方在于阈值被设置为 `0`，也就是说：大于 0 显示上箭头，小于 0 显示下箭头。它放在这里的意义，是告诉读者环比最重要的信息不是绝对值大小，而是方向。看到箭头，读者第一眼就知道是增长还是下降。

为什么环比特别适合图标集？因为这类指标的阅读重点是“趋势”。

- 上箭头：比上月增长
- 下箭头：比上月下降
- 中性图标：基本持平

对业务阅读者来说，这比一串百分比更直观。

### 2. 只显示箭头还不够，最好再加正负颜色

[插图：将大于0的环比值设置为绿色字体]

这张图展示的是另一层强化：给大于 0 的数值设置绿色字体。读者需要关注的是，“条件格式”不只是能加箭头，也能继续叠加字体颜色规则。它之所以放在这里，是因为单纯的箭头解决了方向问题，而颜色会进一步强化涨跌感知。上升用绿色、下降用红色，读者扫一眼整列就能抓住重点。

实际使用中，这种搭配特别适合报表：

- 图标解决“向上还是向下”
- 颜色解决“读者情绪判断”
- 百分比数字保留精确信息

三层叠加，阅读效率会比单纯数字高很多。

### 3. 成交金额适合用数据条，看规模最直观

[插图：为成交金额列添加数据条效果]

这张图应该放在金额分析部分，因为它展示的是“成交金额”这一类绝对值指标最适合的表达方式。图里高亮的是“条件格式 -> 数据条”的入口，以及应用后在金额列里形成的横向条形。读者要重点看的是：数据条会把数值大小直接转成长度差异，所以谁的成交额高、差距有多大，一眼就能看出来。

在这份案例里，像广州、苏州、杭州这几个城市，金额数据条明显更长；深圳、重庆、西安则相对更短。  
这就是数据条的价值：不用读完整数字，也能先建立规模感。

### 4. 占比更适合用色阶，而不是数据条

产品成交情况区块里，“占比”这列没有继续用数据条，而是用了色阶。这其实很合理。

原因在于：

- 金额是绝对值，更适合看量级差异
- 占比是结构值，更适合看高低分布

对比来看：

| 指标类型 | 更适合的条件格式 | 重点表达什么 |
| --- | --- | --- |
| 成交金额 | 数据条 | 规模和差距 |
| 占比 | 色阶 | 相对高低和结构分布 |
| 环比 | 图标集 + 字体颜色 | 方向和趋势 |

博客里讲这一点很重要，因为很多人并不是不会点条件格式，而是不知道不同指标该配什么样的视觉编码。

### 5. 自定义数字格式：让环比更像“报表”，而不是“原始数值”

[插图：设置环比列的自定义数字格式]

这张图展示的是“设置单元格格式 -> 数字 -> 自定义”的位置，以及实际使用的格式代码。它适合放在条件格式后面，因为这一步不是在改变趋势逻辑，而是在优化数值呈现方式。读者这里最该关注的是：正值、负值和异常值可以分别定义显示规则，而且这些规则只影响显示，不影响单元格真实值。

例如环比列里常见的格式可以写成：

```text
[Color10]0%▲;[Red]-0%▼
```

如果还要兼容文字提示，也可以做三段式控制。

这一步的核心认知是：**Excel 里的值和显示样式，是两层东西。**

也就是说：

- 看到 `-17%▼`，不代表单元格里存的是文本
- 公式继续引用它时，仍然使用真实数值
- 你只是让报表对读者更友好，而不是改了底层逻辑

## 八、从结果反推公式：这张报表到底说明了什么

## 九、实操中最容易踩的坑

这类报表最常见的问题，不是不会写函数，而是细节没处理好。

### 1. `SUMIF` 和 `SUMIFS` 参数顺序写反

这是高频错误。

```excel
=SUMIF(条件范围,条件,求和范围)
=SUMIFS(求和范围,条件范围1,条件1,...)
```

两者最容易混的，就是求和范围的位置。  
`SUMIF` 在最后，`SUMIFS` 在最前。

### 2. 月份参数没锁死

像 `$H$4` 这种全表控制参数，一定要锁列也锁行。  
否则公式一拖，月份筛选条件就会漂移，整张表的计算逻辑会一起错掉。

### 3. 客单价被当成普通可汇总指标

这是非常典型的逻辑错误。

错误思路：

- 直接对客单价列求和
- 直接对客单价做条件求和

正确思路：

- 先汇总金额
- 再汇总客户数
- 最后做除法

### 4. 没有上月数据时，不做容错

像南京、1期、3期、9期这类位置，在部分月份没有可比数据。  
如果没有 `IFERROR`，报表就会出现错误值，视觉上非常割裂。

### 5. 把显示样式误以为真实值

无论是条件格式还是自定义数字格式，都只是“显示层”。  
它们可以让 `0.93` 显示成 `93%▲`，也可以让 `-0.17` 显示成 `-17%▼`，但单元格底层存的仍然是数值本身。

如果只是把这张表做出来，你学到的是几个函数；但如果你把它看成一个完整案例来理解，你掌握的其实是一整套 Excel 报表方法。

这份案例至少练到了下面几件事：

- 能从明细表里提炼出报表需要的字段和辅助维度
- 能用数据验证做出真正可用的筛选控制器
- 能用 `SUMIF`、`SUMIFS` 把单条件和多条件汇总串起来
- 能理解为什么跨表引用和绝对引用是动态报表的基础
- 能正确处理环比、客单价和无数据场景
