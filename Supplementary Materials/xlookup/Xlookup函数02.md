### XLOOKUP 函数介绍

XLOOKUP 是 Microsoft Excel 中一个功能强大的现代查找函数，于2019年随 Office 365 推出，旨在替代传统的 VLOOKUP 和 HLOOKUP 函数。它可以在一个范围中查找值，并返回另一个范围中对应位置的值。

**基本语法**

XLOOKUP 函数的语法为：`=XLOOKUP(查找值, 查找数组, 返回数组, [未找到时返回值], [匹配模式], [搜索模式])`

其中，前三个参数为必需参数，后三个为可选参数。

**主要功能特点**

XLOOKUP 支持多种查找模式，包括精确匹配和近似匹配。它还可以处理通配符搜索、正则表达式（regex）、反向搜索，以及垂直或水平范围的查找。函数能够从同一匹配记录中返回多个值，只需一个公式即可完成，这在传统查找函数中很难实现。当 XLOOKUP 找不到匹配项时，默认返回 #N/A 错误，但用户可以通过设置"未找到时返回值"参数来自定义返回值。

**相比 VLOOKUP 的优势**

XLOOKUP 克服了 VLOOKUP 的多个局限性。VLOOKUP 只能从左向右查找，且列索引号是固定的，而 XLOOKUP 可以向左或向右查找，返回数组的位置不受限制。此外，XLOOKUP 不需要指定返回列的索引号，使用起来更加直观，不易出错。它还默认执行精确匹配，这与 VLOOKUP 的默认近似匹配不同，使结果更可预测。

**应用场景**

XLOOKUP 适用于各种数据查找场景，包括日常的数据匹配、报表制作、财务分析等。无论是简单的单值查找，还是复杂的多条件查找，XLOOKUP 都能高效完成。



1. ## **XLOOKUP** 函数的完整语法：

```
=XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found], [match_mode], [search_mode])
```

| 参数         | 名称         | 是否必填 | 说明                                                        |
| ------------ | ------------ | -------- | ----------------------------------------------------------- |
| lookup_value | 查找值       | 必填     | 你要找的内容                                                |
| lookup_array | 查找范围     | 必填     | 在哪一列/行里找                                             |
| return_array | 返回范围     | 必填     | 找到后返回哪一列/行的值                                     |
| if_not_found | 找不到时的值 | 可选     | 默认返回 #N/A 错误                                          |
| match_mode   | 匹配模式     | 可选     | 0=精确匹配（默认），-1=近似匹配（小于），1=近似匹配（大于） |
| search_mode  | 搜索模式     | 可选     | 1=从头到尾（默认），-1=从尾到头（倒序）                     |

### 案例一、竖向查找

**场景：** 在一列数据中，根据姓名查找对应的销售额。这是 XLOOKUP 最基础的用法。

![案例一、竖向查找](.\images\案例一、竖向查找.png)

**查找目标：** 小陈的销售额 → 结果：**60**

**公式：**
```
=XLOOKUP(F2, A2:A13, B2:B13)
```

**参数解析：**
- `F2` → 查找值，即"小陈"所在单元格
- `A2:A13` → 查找范围，姓名列（竖向一列）
- `B2:B13` → 返回范围，销售额列

XLOOKUP 会在 A 列中从上往下逐一比对，找到"小陈"后，返回 B 列对应行的值 60。与 VLOOKUP 相比，XLOOKUP 无需指定列号，查找列和返回列完全解耦，更加灵活。

### 案例二、横向查找

**场景：** 数据是横向排列的（姓名在第一行，销售额在第二行），根据姓名查找销售额。

![案例二、横向查找](E:\桌面文件\xlookup\images\案例二、横向查找.png)

**查找目标：** 小陈的销售额 → 结果：**60**

**公式：**
```
=XLOOKUP(A23, B18:M18, B19:M19)
```

**参数解析：**
- `A23` → 查找值"小陈"
- `B18:M18` → 查找范围，姓名所在的**横向一行**
- `B19:M19` → 返回范围，销售额所在的**横向一行**

这正是 XLOOKUP 相较于 VLOOKUP 的核心优势之一——**天然支持横向查找**，而 VLOOKUP 只能竖向查找，横向需要借助 HLOOKUP。XLOOKUP 一个函数同时覆盖两种方向。



### 案例三、反向查找

**场景：** 返回列在查找列的**左侧**（或上方），即"反向"查找。传统 VLOOKUP 无法向左查找，但 XLOOKUP 完全没有方向限制。

![案例三、反向查找](E:\桌面文件\xlookup\images\案例三、反向查找.png)



**查找目标：** 已知姓名"小王"，查找其工号 → 结果：**A003**

**公式：**
```
=XLOOKUP(A47, C31:C42, B31:B42)
```

**参数解析：**
- `A47` → 查找值"小王"，位于姓名列（C列）
- `C31:C42` → 查找范围，姓名列
- `B31:B42` → 返回范围，工号列（位于姓名列的**左边**）

VLOOKUP 要求返回列必须在查找列右侧，否则需要辅助列或复杂嵌套。XLOOKUP 直接指定任意位置的返回范围，轻松实现反向查找。

### 案例四、整行查找

**场景：** 查找某个值，同时返回该行中**多列**的数据（一次性返回整行）。

![案例四、整行查找](E:\桌面文件\xlookup\images\案例四、整行查找.png)

**公式（截图中可见）：**
```
=XLOOKUP(F33, B31:B42, C31:D42)
```

**参数解析：**
- `F33` → 查找值"A004"（工号）
- `B31:B42` → 查找范围，工号列
- `C31:D42` → 返回范围，**姓名列+销售额列**（两列同时返回）

**结果：** 返回小吴、120 两个值，自动溢出填充到相邻单元格。

这是 XLOOKUP 的**数组溢出特性**：return_array 可以是多列（或多行），函数会自动将所有匹配列的值一次性输出，无需写多个公式。这在 Excel 365/2021 的动态数组环境下尤为强大。

### 案例五、整列查找

**截图公式：**
```
=XLOOKUP(F33:F37, C31:C42, D31:D42)
```

![案例五、整列查找](E:\桌面文件\xlookup\images\案例五、整列查找.png)



**查找目标（右侧）：** 批量查找小张、小李、小蒋、小苏、小马的销售额

**参数拆解：**

| 参数位置     | 实际内容  | 说明                                                         |
| ------------ | --------- | ------------------------------------------------------------ |
| lookup_value | `F33:F37` | **查找值是一个区域**，包含5个姓名：小张、小李、小蒋、小苏、小马 |
| lookup_array | `C31:C42` | 在姓名列中查找                                               |
| return_array | `D31:D42` | 返回对应的销售额列                                           |

**结果：** 一个公式同时输出5行结果，向下自动溢出 → 100、130、300、160、260

**核心要点：** lookup_value 设为一个范围（F33:F37）而非单个单元格，XLOOKUP 会对区域内每一个值逐一匹配，配合 Excel 365 的**动态数组**特性一次性溢出所有结果。截图顶部工具栏也可以看到溢出结果被识别为 `{"小张";"小李";"小蒋";"小苏";"小马"}`，这正是动态数组的体现。

### 案例六、指定错误值

![案例六、指定错误值](E:\桌面文件\xlookup\images\案例六、指定错误值.png)

**截图公式：**

```
=XLOOKUP(F32, C31:C42, D31:D42, "未找到")
```

**查找目标：** 查找"小111"的销售额 → 数据中不存在此人

**参数拆解：**

| 参数位置     | 实际内容   | 说明                            |
| ------------ | ---------- | ------------------------------- |
| lookup_value | `F32`      | 查找值"小111"                   |
| lookup_array | `C31:C42`  | 在姓名列中查找                  |
| return_array | `D31:D42`  | 返回销售额列                    |
| if_not_found | `"未找到"` | **第4参数**，找不到时返回此文字 |

**结果：** 显示 **"未找到"**，而非 `#N/A` 错误

**核心要点：** 截图中编辑栏清晰显示第4参数 `"未找到"` 被加粗高亮（Excel 提示当前光标所在参数），说明此时第4参数正在被编辑。过去需要在外层套 `IFERROR()` 才能实现，XLOOKUP 将错误处理内置到函数本身，写法更简洁。`if_not_found` 可以填入任意文字、数字或空字符串 `""`。



### 案例七、多条件查找

![案例七、多条件查找](E:\桌面文件\xlookup\images\案例七、多条件查找.png)

**截图公式：**
```
=XLOOKUP(A72&B72, F70:F84&G70:G84, H70:H84)
```

**数据源（右侧15行）：**

| 姓名 | 产品     | 销售额 |
| ---- | -------- | ------ |
| 小张 | 水果     | 310    |
| 小李 | 水果     | 500    |
| 小陈 | 水果     | 60     |
| 小张 | 健身器械 | 50     |
| 小陈 | 健身器械 | 140    |
| 小张 | 保健品   | 260    |
| 小陈 | 保健品   | 240    |
| …    | …        | …      |

**查找目标：** 小陈 + 健身器械 的销售额 → 结果：**140**

**参数拆解：**

| 参数位置     | 实际内容          | 说明                                             |
| ------------ | ----------------- | ------------------------------------------------ |
| lookup_value | `A72&B72`         | 将"小陈"和"健身器械"用 `&` 拼接 → "小陈健身器械" |
| lookup_array | `F70:F84&G70:G84` | 将姓名列和产品列逐行拼接，生成虚拟组合列         |
| return_array | `H70:H84`         | 返回销售额列                                     |

**核心要点：** `&` 拼接是构造多条件查找的核心技巧。XLOOKUP 本身只支持单一 lookup_array，通过 `&` 将多列在内存中合并为一列虚拟数组，再用同样方式拼接查找值，从而实现精确的多条件匹配。无需辅助列，也无需 SUMPRODUCT 等复杂函数。注意：拼接时要确保组合后的字符串不会产生歧义（如"小张水果"和"小张水果"不会误匹配）。

### 案例八、倒序查找

![案例八、倒序查找](E:\桌面文件\xlookup\images\案例八、倒序查找.png)

**截图公式（第5张图编辑栏完整可见）：**

```
=XLOOKUP(A77, F70:F84, H70:H84, , , -1)
```

**数据背景：** 右侧数据表中"小张"出现了3次，分别对应水果（310）、健身器械（50）、保健品（260）。默认从上往下查找会返回第一条 310，而案例要求返回**最后一条** 260。

**参数拆解：**

| 参数位置     | 实际内容  | 说明                           |
| ------------ | --------- | ------------------------------ |
| lookup_value | `A77`     | 查找值"小张"                   |
| lookup_array | `F70:F84` | 在姓名列中查找（含多个"小张"） |
| return_array | `H70:H84` | 返回销售额列                   |
| if_not_found | *(留空)*  | 使用默认值                     |
| match_mode   | *(留空)*  | 使用默认精确匹配               |
| search_mode  | `-1`      | **第6参数**，从尾到头倒序搜索  |

**结果：** 返回最后一个"小张"对应的销售额 → **260**（保健品那行）

**核心要点：** 截图编辑栏中公式写作 `A77,F70:F84,H70:H84,,,-1`，注意第4、5参数之间有**两个连续逗号**，代表跳过这两个可选参数直接指定第6参数。`search_mode=-1` 的典型应用场景是流水账查最新记录、查最后一次操作等。

### 案例九、匹配模式

![案例九、匹配模式](E:\桌面文件\xlookup\images\案例九、匹配模式.png)



**截图公式：**

```
=XLOOKUP(D90, A88:A91, B88:B91, , -1)
```

**数据源（左侧提成区间表）：**

| 销售额下限 | 提成比例 |
| ---------- | -------- |
| 50         | 1%       |
| 100        | 2%       |
| 200        | 3%       |
| 400        | 4%       |

**查找目标：** 销售额 310，对应提成比例 → 结果：**3%**

**参数拆解：**

| 参数位置     | 实际内容  | 说明                                      |
| ------------ | --------- | ----------------------------------------- |
| lookup_value | `D90`     | 查找值 310                                |
| lookup_array | `A88:A91` | 在销售额下限列中查找                      |
| return_array | `B88:B91` | 返回提成比例列                            |
| if_not_found | *(留空)*  | 默认                                      |
| match_mode   | `-1`      | **第5参数**，精确匹配或返回下一个更小的值 |

**结果：** 310 不在表中，match_mode=-1 找到不超过 310 的最大值 → **200**，返回对应的 **3%**

**核心要点：** 注意这里 `-1` 是**第5参数 match_mode**（匹配模式），而案例八的 `-1` 是**第6参数 search_mode**（搜索模式），两者位置不同含义完全不同。截图编辑栏 `B88:B91,,-1` 中只有一个逗号间隔，说明跳过的是第4参数 if_not_found，直接指定第5参数。使用近似匹配时，**数据表必须升序排列**，否则结果不可靠。

## **XLOOKUP 核心优势总结**

相比 VLOOKUP/HLOOKUP，XLOOKUP 在以上 9 个案例中体现出的核心进步在于：它同时支持横向与竖向查找，彻底取消了"返回列必须在右侧"的限制，内置错误处理参数，支持批量和整行返回，以及通过 search_mode 和 match_mode 两个参数覆盖了倒序查找和近似匹配等所有高级场景。在 Excel 365 及 2021 版本中，XLOOKUP 可以完全替代 VLOOKUP、HLOOKUP 和大多数 INDEX+MATCH 组合。
